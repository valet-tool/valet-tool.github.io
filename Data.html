The columns in the above sample .csv file are described below:

  1. Counter: This is a simple incremental counter for each record.
  2. Total Power Consumed: The amount of energy the tactic used. This is computed by determining the average energy usage for the system during the tactic's execution, and then finding this difference vs. the system's baseline (when the system is not executing any tasks).
  3. First Started Timestamp: This is the time the tactic started. This is recorded for any time series analysis that the user may want to conduct.
  4. Latency: The amount of time necessary to complete the tactic.
  5. Server Number: Which of the various servers the file was downloaded from (see below for referenced server)
  6. Tactic Number: The tactic that was executed (see below for referenced tactic)

Executed tactics

The following tactic numbers correlate with the tactic operations in the generated .csv output file:

  1. Download file: VALET first downloads a compressed LibreOffice tar.gz 224.8MB file that is hosted on servers around the world. It was selected in part due to its existing widespread presence on the Internet. It is also an adequate file size to produce sufficient tactic and cost latency values. VALET downloads this file from nine servers located in countries such as Canada, Korea, Tanzania, Netherlands, Uruguay, Costa Rica, Brazil, Hungary, and Italy. Downloading this file from several locations created a diverse set of real-world tactic latency volatility, which was created by not only the physical distances between the devices, but also the varying network traffic that each operation would encounter as encounters. This emulates the tactic of performing a communication or file transmission operation.
  2. Extract the file's contents: The collected file is then unzipped using the tar -xzf command, with all contents being extracted. This emulates the tactic of performing a simple file i/o operation.
  3. Perform Grep operation: A grep operation performs a simple keyword search on the extracted file contents. grep was selected since it searches through each of the file within the directory for a specific pattern. This emulated the tactic of performing a simple function or system task.
  4. Compress the file: The file is then re-compressed using the GNU tar command. This operation provides an additional tactic example of a file i/o operation.
  5. File deletion: At the conclusion of the cycle, the file is deleted so that the process is ready to begin again.
